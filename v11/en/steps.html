Import projects

When you click Import, two projects will be created in your workspace: An application named ExampleFileInputWholeFile and a shared library named FileCSV.

ExampleFileInputWholeFile provides a single message flow named FileInputWholeFile which uses a FileInput node to receive an input XML file and propagate the data downstream through the flow. The XML data is transformed into a comma separated output format and written to an output file using a FileOutput node.

Once imported using the link on the right, the provided projects are ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Import

Prepare an output directory, create an integration node and deploy the message flow

Prepare the scenario by executing the following steps:

  1. The message flow's FileInput node will attempt to read input files from the directory C:\temp\FileInput. Create this directory if it does not already exist, or edit the FileInput node in the message flow to read from a different directory suitable for your runtime environment.
  2. The message flow's FileOutput node will attempt to write output files to the directory C:\temp\FileOutput. Create this directory if it does not already exist, or edit the FileOutput node in the message flow to write to a different directory suitable for your runtime environment.
  3. Next, create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  4. Back in the Toolkit, drag and drop deploy the provided BAR file named FileInputWholeFile.bar which you will find inside the ExampleFileInputWholeFile application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open FileInputWholeFile.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Copy the provided input message from <Integration Toolkit workspace>/ExampleFileInputWholeFile/WholeFileInput.xml to the input directory that you created on your file system. The file will be processed by the message flow and then deleted.
    • In the Flow Exerciser toolbar, click the View Path icon to highlight the message path through the message flow. You can click on any highlighted connection to see the message data that passed through that connection.
    • WholeFileInput.xml contains this data:
      <Message xmlns="http://www.ibm.com/ace/dev/FileXML">
      <Field1>United Kingdom</Field1>
      <Field2>USA</Field2>
      <Field3>Germany</Field3>
      <Field4>France</Field4>
      <Field5>India</Field5>
      </Message>
    • The flow should create an output file named C:\temp\FileOutput\WholeFileOutput.txt which contains this data:
      United Kingdom,USA,Germany,France,India

This tutorial has shown the FileInput node parsing the whole content of a file.
